# HOW_I_DID_IT - Level 2 (Track B)

## What I Did (Step by Step)

1. Installed dependencies and verified the repo setup.
2. Ran the LPI sandbox test client using:
   - `npm run test-client`
3. Captured the output as proof in:
   - `submissions/sanskriti/test-client-output.txt`
4. Read the SMILE methodology structure from the sandbox output (especially `smile_overview`) and cross-checked phase details.
5. Reviewed multiple case studies from `get_case_studies` and selected at least two to map into a sports context.
6. Wrote a one-page analysis describing how each SMILE phase can be applied in sports.
7. Added explicit references to case studies used and explained why those lessons transfer to sports.

## Problems I Hit and How I Solved Them

- Problem: There was no direct sports case study in the bundled case-study file.
  - Solution: I used analogical transfer from similar high-stakes domains (healthcare and manufacturing), then mapped each pattern to sports operations and athlete management.

- Problem: It was easy to drift into generic "AI in sports" writing.
  - Solution: I forced the write-up to stay phase-by-phase using the SMILE structure so each section stays concrete and tied to methodology.

## What I Learned

- SMILE is most useful when used as a decision framework, not just a data framework.
- The MVT concept is practical for sports teams because it reduces risk and prevents oversized projects.
- Ontology work (shared definitions) is as important as model accuracy when many stakeholders (coaches, physios, analysts) must trust and use the system.
- Explainability is critical in sports because decisions affect health, careers, and competitive outcomes.
